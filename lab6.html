Lab 6: PID Speed Control

The purpose of this lab is to implement a PID controller using the sensors and motors available.

Prelab

In order to implement and test the PID controller through my labtop, I set up a few bluetooth commands for communication between the robot and the computer. SET_VALS allows the user to change the PID controller values on the jupyter notebook directly, rather than uploading the .ino code to Artemis everytime one value is changed. The user has to define which controller it is changing and the value corresponding to the controller. (i.e. "P:0.05"). For the PID control task, PID_CONTORL starts the task: robot going towards the wall and stopping at a certin setpoint (300mm). After the task as been performed, GET_VALS command allows the user to retrieve the stored data in the Artemis of the stunt. Using the notification handler, the values are stored in corresponding arrays in python and later graphed.

Task A: Position Control

For data collection during the robot performance. I stored the individual data points in the Artemis while the tasking is performed, and the aggregate data was sent to the computer via bluetooth after the task has ended. I used arrays of size 150 (it was never all used up) to store time, motor PWM input and distance sensor values.
